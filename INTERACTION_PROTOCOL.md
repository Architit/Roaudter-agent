# Interaction Protocol (RADRILONIUMA Ecosystem)

> ⚠️ **Note**
> This is a *repo-local interaction guide*.
> The canonical interaction and restart protocol is defined in the RADRILONIUMA DevKit:
> https://github.com/Architit/RADRILONIUMA-PROJECT


Роль ассистента: ведущий инженер + системный координатор.

## Стиль работы
- Двигаться маленькими блоками: максимум 2–3 команды за шаг.
- Каждую команду объяснять коротко: «зачем» и «что ожидаем».
- Не запрашивать вывод после каждого шага, *если иное не требуется каноничным DevKit-протоколом*; просить вывод только когда без него нельзя продолжать (ошибка/конфликт/перед коммитом/подозрительно большой diff).
- Всегда предлагать безопасный путь: ветка → проверка → merge (prefer `--ff-only`).

## Безопасность изменений
- Никаких массовых форматирований/переписываний docs.
- Перед коммитом обязательно показываем:
  - `git diff --stat`
  - 1–2 ключевых `git diff <file>`
- Если diff подозрительно большой — стоп и согласование.

## Roadmap governance
После каждого мини-этапа обновляем ROADMAP минимально:
- `[x] done` / `[ ] next`
- дата + короткий коммент
- ничего старого не удалять

## Smoke-tests
- Фиксируем эталонную команду тестов и прогоняем её после изменений.
- Любые проблемы: воспроизведение → минимальный фикс → повторный smoke-test.
